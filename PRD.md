# 📘 Product Requirements Document (PRD)

## DY Carbon  
### 산업용 카본·그래파이트 B2B 웹 플랫폼

---

## 0. 제품 목표 정의

### 서비스 목적
DY Carbon은 산업용 카본 및 그래파이트 제품 정보를 구조화하여 제공하고,  
고객이 **제품 이해 → 비교 → 견적 문의**까지 빠르게 도달할 수 있도록 돕는  
B2B 중심 웹 플랫폼이다.

기존 전화·이메일 중심의 비효율적인 문의 방식을 개선하고,  
제품 탐색과 문의를 하나의 사용자 흐름으로 통합하는 것이 핵심 목표다.

---

## 1. Phase 1 – MVP (수업 내 구현 목표)

### 1.1 Phase 1 목표
- 3일 수업 기간 내 **완주 가능한 기능 범위**
- B2B 서비스의 핵심인 **문의 전환 흐름**을 실제로 구현
- 인증, CRUD, AI 연동을 모두 경험할 수 있는 최소 단위 MVP

---

### 1.2 핵심 사용자 시나리오

1. 사용자는 Google 계정으로 로그인한다.
2. 제품 리스트를 조회한다.
3. 특정 제품의 상세 정보를 확인한다.
4. 제품을 기준으로 간단한 텍스트 문의를 작성한다.
5. 문의 내용에 대해 AI 응답을 확인한다.
6. 문의와 AI 응답이 저장된다.

---

### 1.3 인증 (Auth)

#### 구현 범위
- Google Social Login 단일 방식
- OAuth 기반 사용자 식별

#### 제외 항목
- 이메일/비밀번호 로그인
- 관리자 권한 분기
- 세부 권한 정책

---

### 1.4 데이터 모델 (Phase 1)

#### User
| 필드 | 설명 |
|---|---|
| id | 내부 사용자 ID |
| google_id | Google OAuth ID |
| name | 사용자 이름 |
| email | 이메일 |

---

#### Product
| 필드 | 설명 |
|---|---|
| id | 제품 ID |
| name | 제품명 |
| category | Carbon / Graphite |
| summary | 한 줄 요약 |
| description | 상세 설명 |
| created_at | 등록 시점 |

---

#### Inquiry
| 필드 | 설명 |
|---|---|
| id | 문의 ID |
| user_id | 작성자 |
| product_id | 문의 대상 제품 |
| content | 문의 텍스트 |
| ai_response | AI 응답 결과 |
| created_at | 문의 시점 |

---

### 1.5 기능 명세 (CRUD)

#### 제품
- 제품 리스트 조회
- 제품 상세 조회  
※ Phase 1에서는 읽기 전용

---

#### 문의
- 문의 작성
- 내가 작성한 문의 목록 조회
- 문의 삭제

---

### 1.6 AI 기능 (Phase 1)

#### 목적
- 제품 이해를 돕는 보조 도구
- AI 연동 및 결과 저장 구조 검증

#### 기능
- 사용자가 입력한 문의 텍스트를 기반으로 AI 응답 생성
- 응답 예시:
  - 문의 내용 요약
  - 일반적인 사용 용도 설명

#### 저장
- AI 응답은 Inquiry 데이터에 함께 저장

---

### 1.7 페이지 구성

1. 메인 페이지
   - 서비스 소개 문구
   - 제품 리스트로 이동 버튼

2. 제품 리스트 페이지
   - 카드형 제품 목록
   - 제품명 및 요약 정보 표시

3. 제품 상세 페이지
   - 제품 설명
   - 문의하기 버튼

4. 문의 작성 페이지
   - 텍스트 입력
   - AI 응답 결과 표시

5. 마이페이지
   - 내가 작성한 문의 목록

---

### 1.8 Phase 1 완료 기준

- Google 로그인 가능
- 제품 조회 가능
- 문의 작성 → AI 응답 → 데이터 저장 흐름 구현
- 데모 시나리오 5분 이내 시연 가능

---

## 2. Phase 2 – 고도화 (수업 이후 확장)

### 2.1 제품 데이터 고도화
- 물성 정보 구조화
- 산업 태그 다대다 관계
- 제품 비교 기능

---

### 2.2 문의 기능 확장
- 문의 상태 관리 (접수 / 검토 / 응답)
- 관리자 메모 및 히스토리
- 문의 데이터 분석

---

### 2.3 AI 기능 확장
- 사용 목적 기반 제품 추천
- 산업별 수요 분석
- 관리자용 AI 요약 리포트

---

### 2.4 실시간 및 시스템 기능
- 알림 시스템
- 사용자 ↔ 관리자 실시간 채팅
- 파일 및 이미지 업로드

---

### 2.5 대시보드 및 시각화
- 문의 추이 차트
- 인기 제품 분석
- 산업별 수요 흐름 시각화

---

### 2.6 외부 연동
- CRM 연동
- 유료 AI API 활용
- 이메일 및 메신저 알림

---

## 3. Phase 요약

| 구분 | Phase 1 (MVP) | Phase 2 (고도화) |
|---|---|---|
| 인증 | Google 로그인 | 권한 관리 |
| CRUD | 기본 텍스트 중심 | 구조화 데이터 |
| AI | 단순 응답 | 추천·분석 |
| 데이터 활용 | 저장 중심 | 분석 자산화 |
| 목적 | 완주 경험 | 실서비스 확장 |

---

## 4. 시니어 개발자 관점 요약

Phase 1은 “완성도 높은 작은 서비스”가 아니라  
**“명확한 사용자 흐름을 끝까지 구현한 MVP”를 목표로 한다.  
Phase 2는 Phase 1에서 만든 구조를 기반으로 확장한다.
